## Why fallow?

Linters enforce style. Formatters enforce consistency. Fallow enforces relevance.

AI writes code. Nobody deletes it. Fallow finds what's left behind. Dead code, circular dependencies, duplicated logic, and complexity hotspots slow builds, bloat bundles, and make codebases harder to navigate. Agents generate code but don't clean up after themselves. They can't: **static analysis requires building and traversing a <Tooltip tip="A directed graph where nodes are files and edges are import/export relationships between them">module graph</Tooltip>**, which is fundamentally different from reading files in a context window.

Fallow builds the full import graph, traces re-export chains through <Tooltip tip="Index files (typically index.ts) that re-export symbols from other modules for cleaner imports">barrel files</Tooltip>, detects dependency cycles, finds duplicated logic, and surfaces complexity hotspots. Deterministically, exhaustively, with zero configuration.

## Get started

<Tabs>
  <Tab title="Agents (CLI + MCP)">
    Agents can run fallow directly from the command line. No MCP required:

    ```bash
    npx fallow check --format json
    npx fallow fix --dry-run --format json
    ```

    For structured tool calling via MCP, add to your agent's config:

    ```json
    {
      "mcpServers": {
        "fallow": {
          "command": "fallow-mcp"
        }
      }
    }
    ```

    Your agent can now call `analyze`, `check_changed`, `find_dupes`, `fix_preview`, `fix_apply`, and `project_info` as structured tools. [Full agent integration guide](/integrations/mcp)
  </Tab>
  <Tab title="Humans (CLI + VS Code)">
    ```bash
    npm install -g fallow
    fallow check
    fallow fix --dry-run
    ```

    Install the [VS Code extension](https://marketplace.visualstudio.com/items?itemName=fallow-rs.fallow-vscode) for real-time diagnostics and Code Lens. [Quick start guide](/quickstart)
  </Tab>
  <Tab title="CI (GitHub Actions)">
    ```yaml
    - uses: fallow-rs/fallow@v1
      with:
        format: sarif
    ```

    Upload SARIF to GitHub Code Scanning for inline annotations on PR diffs. [CI integration guide](/integrations/ci)
  </Tab>
</Tabs>
