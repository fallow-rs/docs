## Why fallow?

Linters enforce style. Formatters enforce consistency. Fallow enforces relevance.

AI writes code. Nobody deletes it. Fallow finds what's left behind. Dead code, circular dependencies, duplicated logic, and complexity hotspots slow builds, bloat bundles, and make codebases harder to navigate. Agents generate code but don't clean up after themselves. They can't: **static analysis requires building and traversing a <Tooltip tip="A directed graph where nodes are files and edges are import/export relationships between them">module graph</Tooltip>**, which is a different problem from reading files in a context window.

Fallow builds the full import graph, traces re-export chains through <Tooltip tip="Index files (typically index.ts) that re-export symbols from other modules for cleaner imports">barrel files</Tooltip>, detects dependency cycles, finds duplicated logic, and flags complexity hotspots. Deterministic, exhaustive, zero configuration.

## Get started

<Tabs>
  <Tab title="Agents (CLI + MCP)">
    Agents can run fallow directly from the command line. No MCP required:

    ```bash
    npx fallow --format json                # All analyses
    npx fallow dead-code --format json      # Dead code only
    npx fallow dupes --format json          # Duplication only
    npx fallow health --format json         # Complexity hotspots
    npx fallow fix --dry-run --format json  # Preview auto-fixes
    ```

    For structured tool calling via MCP, add to your agent's config:

    ```json
    {
      "mcpServers": {
        "fallow": {
          "command": "fallow-mcp"
        }
      }
    }
    ```

    Your agent can now call `analyze`, `check_changed`, `find_dupes`, `check_health`, `fix_preview`, `fix_apply`, and `project_info` as structured tools. [Full agent integration guide](/integrations/mcp)
  </Tab>
  <Tab title="Humans (CLI + VS Code)">
    ```bash
    npm install -g fallow
    fallow              # All analyses
    fallow fix --dry-run
    ```

    Install the [VS Code extension](https://marketplace.visualstudio.com/items?itemName=fallow-rs.fallow-vscode) for real-time diagnostics and Code Lens. [Quick start guide](/quickstart)
  </Tab>
  <Tab title="CI">
    <Tabs>
      <Tab title="GitHub Action">
        ```yaml
        - uses: fallow-rs/fallow@v2
          with:
            format: sarif
        ```
      </Tab>
      <Tab title="GitLab CI">

          extends: .fallow

      </Tab>
      <Tab title="Other CI">
        ```yaml

        ```
      </Tab>
    </Tabs>

    Supports SARIF upload (GitHub Code Scanning), Code Quality reports (GitLab MR annotations), and `--ci` mode for any pipeline. [CI integration guide](/integrations/ci)
  </Tab>
</Tabs>
