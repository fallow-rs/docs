## Why fallow?

Dead code and duplication increase bundle sizes, slow CI, and make codebases harder to navigate. AI-assisted development accelerates this — agents generate code but rarely suggest deletions.

<CardGroup cols={3}>
  <Card title="Zero config" icon="bolt">
    Works out of the box. 84 built-in plugins auto-detect Next.js, Vite, Jest, Tailwind, and 75+ more.
  </Card>
  <Card title="Sub-second" icon="gauge-high">
    Rust-native with Oxc parser and rayon parallelism. Analyzes 5,000+ files in under 200ms, using 3-15x less memory than knip.
  </Card>
  <Card title="One tool, two jobs" icon="layer-group">
    Dead code analysis **and** duplication detection in a single binary. Replace both knip and jscpd.

## Speed comparison

<Tabs>
  <Tab title="vs knip (dead code)">
    | Project | Files | fallow | knip v5 | knip v6 | vs v5 | vs v6 |
    |:--------|------:|-------:|--------:|--------:|------:|------:|
    | [zod](https://github.com/colinhacks/zod) | 174 | **23ms** | 590ms | 308ms | **26.1x** | **13.6x** |
    | [fastify](https://github.com/fastify/fastify) | 286 | **22ms** | 804ms | 236ms | **36.2x** | **10.6x** |
    | [preact](https://github.com/preactjs/preact) | 244 | **24ms** | 799ms | —† | **33.9x** | — |
    | synthetic (1,000 files) | 1,000 | **45ms** | 380ms | 196ms | **8.5x** | **4.4x** |
    | synthetic (5,000 files) | 5,000 | **201ms** | 646ms | 340ms | **3.2x** | **1.7x** |

    <Info>
      Speedup narrows on larger projects as analysis time dominates over startup: 26-36x on real-world projects vs knip v5 (10-14x vs v6), 3-9x on 1,000+ file projects. Fallow stays sub-second even at 5,000 files. † knip v6 excluded for preact due to a v6 regression on this project.
    </Info>
  </Tab>
  <Tab title="vs jscpd (duplication)">
    | Project | Files | fallow | jscpd | Speedup |
