AI coding tools are good at producing code quickly. They are much worse at guaranteeing whole-codebase correctness.

A model can suggest a valid patch while still missing important facts:

- the helper it just wrote already exists elsewhere
- the file it touched is already too complex
- the change crossed a boundary rule
- the deleted function was still runtime-hot
- the new code made an old utility dead
- the safest change would have been editing an existing module instead of adding another one

Fallow exists to supply those missing facts. It does not generate code and does not contain an LLM.

<Info>
  **No AI inside. Built for teams that code with AI.** Fallow is deterministic codebase intelligence that fits around coding agents and human reviewers, not another AI product.
</Info>

## What Fallow contributes to AI-assisted coding

### Static truth

Fallow builds a full module graph and finds:

- unused files, exports, and dependencies
- duplication across files and modules
- complexity hotspots and refactor targets
- architecture boundary violations
- feature flag definitions and guarded branches

### Runtime truth

With Runtime enabled, Fallow also shows:

- which functions actually ran
- which stayed cold
- which changes landed on hot paths
- which branches look stale in production

## Typical workflow

<Steps>
  <Step title="Agent proposes or applies a patch">
    Claude Code, Cursor, Codex, or another coding agent generates or edits code.
  </Step>
  <Step title="Fallow checks the patch or the changed files">
    ```bash
    npx fallow audit --base main --format json
    ```

    The agent (or CI) gets deterministic output: dead code, duplication, complexity, and verdict.
  </Step>
  <Step title="The agent fixes obvious issues">
    The agent applies safe auto-fixes, consolidates duplication, or revises the patch based on structured findings.
  </Step>
  <Step title="Humans review where risk is highest">
    Reviewers spend time on hot-path changes, complexity growth, and boundary drift instead of reading every file equally.
  </Step>
</Steps>

## Why this matters

AI increases throughput, but it also increases review pressure. The bottleneck shifts from writing code to judging it. Fallow helps teams route that attention using evidence.

## What Fallow is not

- Fallow is not an LLM.
- Fallow is not a chatbot.
- Fallow is not a code generator.
- Fallow does not replace review.

It is a deterministic analysis layer that fits around coding agents and human workflows.
