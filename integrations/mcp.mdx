`fallow-mcp` is an MCP server that exposes fallow's analysis capabilities to AI agents. [MCP (Model Context Protocol)](https://modelcontextprotocol.io) is an open standard for connecting AI models to external tools — it lets agents like Claude Code, Cursor, and GitHub Copilot run fallow commands and reason about the results.

<Info>
Tools return JSON, making integration seamless with AI model outputs. Agents can analyze dead code, find duplicates, and apply fixes — all through structured tool calls.
</Info>

## Setup

The MCP server uses stdio transport and wraps the `fallow` CLI binary. Set the `FALLOW_BIN` environment variable to point to the fallow binary (defaults to `fallow` in `PATH`).

<Tabs>
  <Tab title="Claude Code">
    Add to your `.claude/settings.json`:

    ```json
    {
      "mcpServers": {
        "fallow": {
          "command": "fallow-mcp"
        }
      }
    }
    ```
  </Tab>
  <Tab title="Cursor">
    Add to your Cursor MCP settings:

    ```json
    {
      "mcpServers": {
        "fallow": {
          "command": "fallow-mcp"
        }
      }
    }
    ```
  </Tab>
  <Tab title="Other">
    Any MCP-compatible client can connect to `fallow-mcp`. The server uses stdio transport:

    ```bash
    # Start the MCP server directly
    fallow-mcp

    # With a custom fallow binary path
    FALLOW_BIN=/usr/local/bin/fallow fallow-mcp
    ```

    Configure your client to launch `fallow-mcp` as a stdio subprocess.
  </Tab>
</Tabs>

## Available tools

| Tool | Description |
|:-----|:------------|
| `analyze` | Full dead code analysis (`fallow check --format json`) |
| `check_changed` | Incremental analysis of changed files (`fallow check --changed-since`) |
| `find_dupes` | Code duplication detection (`fallow dupes --format json`) |
| `fix_preview` | Dry-run auto-fix preview (`fallow fix --dry-run --format json`) |
| `fix_apply` | Apply auto-fixes (`fallow fix --yes --format json`) |
| `project_info` | Project metadata: plugins, files, entry points (`fallow list --format json`) |

<CodeGroup>
```json Example request
{
  "tool": "analyze",
  "arguments": {
    "production": true,
    "issue_types": ["unused-exports", "unused-files"]
  }
}
```

```json Example response
{
  "issues": [
    {
      "type": "unused-export",
      "file": "src/utils/format.ts",
      "symbol": "formatCurrency",
      "line": 12,
      "severity": "warning"
    }
  ],
  "summary": {
    "total_issues": 1,
    "files_analyzed": 142
  }
}
```
</CodeGroup>

<Tip>
The MCP server wraps the CLI, so all fallow features are available to AI agents — including production mode, baselines, and incremental analysis.
</Tip>

## Architecture

The MCP server is a thin subprocess wrapper — all analysis logic stays in the CLI. The MCP crate only handles protocol framing and argument mapping, built with `rmcp` (official Rust MCP SDK).
