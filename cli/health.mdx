Analyze function complexity across your codebase. Fallow computes cyclomatic and cognitive complexity for every function, then reports those exceeding configurable thresholds.

<Tip>
  Use `--top N` to quickly find the most complex functions without setting thresholds.
</Tip>

```bash
fallow health
```

## Options

### Thresholds

| Flag | Description |
|:-----|:------------|
| `--max-cyclomatic <N>` | Maximum cyclomatic complexity before reporting (default: 20) |
| `--max-cognitive <N>` | Maximum cognitive complexity before reporting (default: 15) |

### Output

| Flag | Description |
|:-----|:------------|
| `-f, --format <FORMAT>` | Output format: `human` (default), `json`, `compact` |
| `--top <N>` | Show only the top N most complex functions |
| `--sort <METRIC>` | Sort by: `cyclomatic` (default), `cognitive`, `lines` |

## Exit codes

| Code | Meaning |
|:-----|:--------|
| `0` | No functions exceed the configured thresholds |
| `1` | One or more functions exceed a threshold |

## Examples

<CodeGroup>
```bash Basic analysis
# Report functions exceeding default thresholds
fallow health
```

```bash Top hotspots
# Show the 10 most complex functions
fallow health --top 10

# Sort by cognitive complexity
fallow health --top 10 --sort cognitive

# Sort by lines of code
fallow health --top 10 --sort lines
```

```bash Custom thresholds
# Stricter thresholds
fallow health --max-cyclomatic 10 --max-cognitive 8

# Only check cognitive complexity
fallow health --max-cyclomatic 999 --max-cognitive 10
```

```bash CI integration
# JSON output for pipelines
fallow health --format json

# Compact output for summaries
fallow health --format compact
```
</CodeGroup>

## Configuration

Configure default thresholds and ignore patterns in your config file:

<CodeGroup>
```jsonc .fallowrc.json
{
  "health": {
    "maxCyclomatic": 20,
    "maxCognitive": 15,
    "ignore": ["**/*.generated.ts", "src/legacy/**"]
  }
}
```

```toml fallow.toml
[health]
maxCyclomatic = 20
maxCognitive = 15
ignore = ["**/*.generated.ts", "src/legacy/**"]
```
</CodeGroup>
