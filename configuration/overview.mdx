<Tip>
Fallow works zero-config for most projects. Only create a config if you need to customize entry points, rules, or ignore patterns.
</Tip>

Fallow works out of the box with zero configuration. For customization, create a config file:

```bash
fallow init          # Creates .fallowrc.json
fallow init --toml   # Creates fallow.toml
```

## Config file formats

Fallow searches for config files in this order:

1. `.fallowrc.json` (JSONC -- comments allowed)
2. `fallow.toml`
3. `.fallow.toml`

## Full example

<CodeGroup>
```jsonc .fallowrc.json
{
  "$schema": "https://raw.githubusercontent.com/fallow-rs/fallow/main/schema.json",
  "entry": ["src/workers/*.ts", "scripts/*.ts"],
  "ignorePatterns": ["**/*.generated.ts", "**/*.d.ts"],
  "ignoreDependencies": ["autoprefixer", "@types/node"],
  "ignoreExports": [
    { "pattern": "src/public-api.ts", "exports": ["*"] }
  ],
  "rules": {
    "unused-files": "error",
    "unused-exports": "warn",
    "unused-types": "off",
    "unresolved-imports": "error"
  },
  "duplicates": {
    "mode": "mild",
    "minTokens": 50,
    "minLines": 5,
    "threshold": 10
  },
  "production": false
}
```

```toml fallow.toml
entry = ["src/workers/*.ts", "scripts/*.ts"]
ignorePatterns = ["**/*.generated.ts", "**/*.d.ts"]
ignoreDependencies = ["autoprefixer", "@types/node"]

[rules]
unused-files = "error"
unused-exports = "warn"
unused-types = "off"
unresolved-imports = "error"

[duplicates]
mode = "mild"
minTokens = 50
minLines = 5
threshold = 10

production = false
```
</CodeGroup>

## Config fields

<Accordion title="entry">
Additional entry point glob patterns. Fallow auto-detects entry points from `package.json` (`main`, `module`, `bin`, `exports`) and framework plugins. Use this for files that aren't auto-detected.

```jsonc
{
  "entry": ["src/workers/*.ts", "scripts/*.ts", "src/cli.ts"]
}
```
</Accordion>

<Accordion title="ignorePatterns">
Glob patterns for files to exclude from analysis entirely.

```jsonc
{
  "ignorePatterns": ["**/*.generated.ts", "**/*.d.ts", "legacy/**"]
}
```
</Accordion>

<Accordion title="ignoreDependencies">
Package names that should always be considered used.

```jsonc
{
  "ignoreDependencies": ["autoprefixer", "@types/node"]
}
```
</Accordion>

<Accordion title="ignoreExports">
Rules for ignoring specific exports. Useful for public API packages.

```jsonc
{
  "ignoreExports": [
    { "pattern": "src/index.ts", "exports": ["*"] },
    { "pattern": "src/utils/*.ts", "exports": ["default"] }
  ]
}
```
</Accordion>

<Accordion title="duplicates">
Configuration for `fallow dupes`:

```jsonc
{
  "duplicates": {
    "mode": "mild",
    "minTokens": 50,
    "minLines": 5,
    "threshold": 10
  }
}
```
</Accordion>

<Accordion title="production">
Enable [production mode](/analysis/production-mode) to focus on code that ships to users.

```jsonc
{
  "production": true
}
```
</Accordion>

<Accordion title="workspaces">
Override configuration for specific workspace packages in a monorepo:

```jsonc
{
  "workspaces": {
    "packages/ui": {
      "entry": ["src/index.ts"],
      "ignoreExports": [
        { "pattern": "src/**/*.tsx", "exports": ["*"] }
      ]
    }
  }
}
```
</Accordion>

<Warning>
Avoid overly broad `ignorePatterns` -- they exclude files from analysis entirely. Use `ignoreExports` for finer control over which exports are considered used.
</Warning>

## JSON Schema

The `$schema` field enables autocomplete and validation in your editor:

```jsonc
{
  "$schema": "https://raw.githubusercontent.com/fallow-rs/fallow/main/schema.json"
}
```

Generate it locally:

```bash
fallow config-schema > schema.json
```
