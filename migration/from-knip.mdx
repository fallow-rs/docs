Fallow provides a one-command migration path from knip. Your existing configuration is automatically translated, so you can start running fallow in minutes.

## Migration workflow

<Steps>
  <Step title="Preview the migration">
    Run `fallow migrate --dry-run` to see the generated config without writing anything:

    ```bash
    fallow migrate --dry-run
    ```

    Fallow searches for: `knip.json`, `knip.jsonc`, `.knip.json`, `.knip.jsonc`, and the `knip` field in `package.json`.

    ```bash title="$ fallow migrate --dry-run"
    Detected knip config: knip.json

    Migrating settings:
      entry          â†’ entry: ["src/index.ts", "src/cli.ts"]
      project        â†’ (inverted to ignorePatterns)
      ignore         â†’ ignorePatterns: ["**/*.generated.ts"]
      ignoreDeps     â†’ ignoreDependencies: ["@types/node"]

    âš  Cannot migrate:
      ignoreBinaries â†’ not applicable in fallow

    Would write: .fallowrc.json
    Run without --dry-run to write the config.
    ```
  </Step>

  <Step title="Generate the config">
    Once you're happy with the preview, run the migration:

    ```bash
    fallow migrate            # Write .fallowrc.json
    fallow migrate --toml     # Or output as fallow.toml
    ```
  </Step>

  <Step title="Review the config">
    Open the generated config and verify the translated entry points, ignore patterns, and plugin mappings. Warnings are shown for any fields that couldn't be migrated automatically.
  </Step>

  <Step title="Run fallow check">
    Run your first analysis and compare results against knip:

    ```bash
    fallow check
    ```
  </Step>
</Steps>

<Tip>
  Run `fallow migrate --dry-run` first to preview the generated config before writing anything to disk.
</Tip>

## What gets migrated

| knip | fallow |
|:-----|:-------|
| `entry` | `entry` |
| `project` | `ignorePatterns` (inverted) |
| `ignore` | `ignorePatterns` |
| `ignoreDependencies` | `ignoreDependencies` |
| `ignoreBinaries` | -- (not applicable) |
| Plugin configs | Closest fallow plugin equivalent |

## Comparison

<Tabs>
  <Tab title="Speed">
    Fallow is a Rust-native standalone binary. No Node.js runtime, no JIT warmup, no garbage collection pauses.

    | Metric | fallow | knip |
    |:-------|:-------|:-----|
    | Speed vs knip v5 | **3--36x faster** | Baseline |
    | Speed vs knip v6 | **2--14x faster** | Baseline |
    | Memory usage | **3--15x less** | Baseline |
    | Runtime dependency | None (standalone binary) | Node.js |
    | CI setup | Download binary | Install Node + npm dependencies |

    Knip v6 is significantly faster than v5, but fallow still leads by 2-14x. On large monorepos the gap widens further because fallow parallelizes across all available cores.
  </Tab>

  <Tab title="Features">
    Fallow ships capabilities that knip does not offer:

    | Feature | fallow | knip |
    |:--------|:-------|:-----|
    | Duplication detection | Built-in (`fallow dupes`) | Not included |
    | SARIF output | Yes | No |
    | Baseline comparison | Yes (`--baseline`) | No |
    | Git-aware analysis | `--changed-since` | No |
    | Production mode | `--production` | No |
    | Auto-fix | Exports + dependencies | Exports + dependencies |
    | Config format | JSONC, JSON, TOML | JSON |
  </Tab>

  <Tab title="Coverage">
    Knip has broader plugin coverage today, but fallow covers the most popular frameworks and allows custom plugins.

    | Metric | fallow | knip |
    |:-------|:-------|:-----|
    | Framework plugins | 84 (30 with config parsing) | 140+ |
    | Dead code issue types | 10 | Comparable |
    | Custom plugins | Yes | Yes |
  </Tab>
</Tabs>

## Key differences

### Plugin coverage

Knip has broader plugin coverage (140+ vs 84). Fallow covers the most popular frameworks. If your framework isn't supported, you can create a custom plugin.

<Info>
  If knip has plugins that fallow doesn't support yet, you can [create a custom plugin](/frameworks/custom-plugins) to fill the gap.
</Info>

### No runtime dependency

Fallow is a standalone binary -- no Node.js runtime required. This means faster CI setup and no dependency conflicts.

### Additional features in fallow

- **Code duplication detection** -- `fallow dupes` with 4 detection modes
- **SARIF output** -- upload to GitHub Code Scanning
- **Baseline comparison** -- `--baseline` for incremental adoption
- **Git-aware analysis** -- `--changed-since` for PR-only checks
- **Production mode** -- `--production` for shipping-code-only analysis
